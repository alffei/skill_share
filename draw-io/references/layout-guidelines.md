# 布局指南

## 1. 分组原则

- 以 AWS Cloud 分组作为最外层
- 按功能单元创建子分组
- 分组以横向排列为基本原则，沿数据流方向放置

### 1.1. 分组层级结构

```text
AWS Cloud (最外层)
├── VPC
│   ├── Public Subnet
│   │   └── ALB, NAT Gateway 等
│   └── Private Subnet
│       └── ECS, RDS 等
├── S3
├── CloudWatch
└── 其他服务
```

## 2. 连接线规则

### 2.1. 线型区分

| 流程类型 | 线型 | 用途 |
|---------|------|------|
| Ingestion Flow | 虚线 | 数据摄入 |
| Query Flow | 实线 | 查询与引用 |
| Control Flow | 点线 | 控制与管理 |

### 2.2. 箭头方向

- 箭头沿数据流动方向指向
- 双向通信使用双向箭头

## 3. 布局原则

### 3.1. 从左到右的流向

```text
[数据源] → [处理] → [存储] → [分析/可视化]
```

### 3.2. 从上到下的流向（备选）

```text
[用户/客户端]
        ↓
[负载均衡器]
        ↓
[应用程序]
        ↓
[数据库]
```

## 4. 可读性保障

- 标签放置在元素附近
- 调整布局避免箭头交叉
- 相关元素分组并就近放置
- 保留适当留白以提升可读性
