# Style Patterns

Use this reference when the topic is concept-heavy, abstract, or likely to confuse non-specialists.

## Fast Structure Choices

- Definition-first: use when the user asks “X 是什么”
- Mechanism-first: use when the user asks “X 怎么工作的”
- Comparison-first: use when readers confuse similar concepts
- Impact-first: use when the core question is “这有什么用”

## Reliable Explanation Frames

### Frame 1: Concept + Mechanism + Meaning

Use for most tech explainers.

1. Say what it is in one sentence
2. Explain the key moving parts
3. Explain why the reader should care

### Frame 2: Problem + Solution + Tradeoff

Use for products, architectures, and new industry ideas.

1. What problem existed before
2. What changed
3. What improved and what remains imperfect

### Frame 3: Misunderstanding + Correction

Use when public discussion is noisy.

1. State the common misunderstanding
2. Correct it simply
3. Explain the real mechanism

## Analogy Rules

- Choose analogies from daily life: office, traffic, kitchen, school, teamwork
- Map only the relevant part of the analogy
- Explicitly say where the analogy stops if precision matters
- Drop the analogy once understanding is established

## Compression Tactics

When the user wants the copy shorter:

- Replace abstract nouns with verbs
- Delete transition filler
- Keep one claim per sentence
- Merge nearby points with the same conclusion
- Move the summary sentence to the top of the section

## Tone Targets

- Plain but not childish
- Professional but not academic
- Dense enough to teach, light enough to finish
